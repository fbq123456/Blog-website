## 修复鼠标互动无效 bug（2026-04-11 第三次修复）

核心 bug：`THREE.Plane` 的 constant 为 0，即相交平面在 z=0，但粒子分布在 z=-10~-140，
`mouseTarget.z=0`，每帧 dist 远超 mouseRadius(20)，排斥力永远为 0。
修复方案：
- `this.plane = new THREE.Plane(new THREE.Vector3(0,0,1), 80)` → 平面定位到 z=-80（粒子中心深度）
- 排斥力改为只计算 XY 平面距离（去掉 Z 差值干扰）
- mouseForce 从 0.05/0.08 → 2.0，mouseRadius 从 15/20 → 25
- returnForce 从 0.03 → 0.02，让排斥效果更持久可见

## 再次修复鼠标距离问题（2026-04-11 第四次修复）

用户反馈鼠标距离和粒子距离很远。

核心 bug：`intersectPlane` 返回布尔值表示是否相交，真正的交点写入 target 参数，但 API 使用方式可能有误。
改为直接用射线方程与 z=-80 平面求交：
```
t = (targetZ - ray.origin.z) / ray.direction.z
mouseTarget = ray.origin + ray.direction * t
```
这样确保 mouseTarget 精确落在粒子所在的 z=-80 平面上，与粒子 XY 坐标可直接比较距离。

同时移除了不再需要的 `this.plane` 属性。
