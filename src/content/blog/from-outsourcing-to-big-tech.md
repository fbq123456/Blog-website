> 没有人脉，没有名校背景，从一个外包公司的前端小白，到大厂的前端开发。这条路走了 3 年，今天第一次把完整经历写出来。

---

## 写在前面

这篇文章不是鸡汤，也不是炫耀。我只是想把一个普通程序员的成长路径记录下来，给同样在迷茫期的朋友一些参考。

每个人都有自己的节奏，不用和别人比。

---

## 第一章：外包的日子

### 入行

2023 年，我从一所普通二本毕业，学了 4 年"计算机科学与技术"——但对编程还是一知半解。

毕业那会儿，简历投了 30 多家，面试通过 3 家，全都是外包公司。最后选了一家离得最近的，月薪 5K。

**我入职时连 Git 都不会用。**

第一天上班，导师丢给我一个 Vue2 的老项目，说：「先看看代码，不懂就问。」

我打开项目，看到 `src` 文件夹里有 200 多个 `.vue` 文件，整个人懵了。

### 外包的真实生活

很多人对外包有偏见，我来说说真实的情况：

**好的方面**：
- 入门门槛低，适合积累经验
- 接触的项目类型多（电商、政务、金融、医疗……）
- 被客户催进度，被迫学会了快速学习和适应

**不好的方面**：
- 技术栈老旧（Vue2 + Element UI + jQuery 混用）
- 没有代码评审，代码质量全靠自觉
- 没有技术分享和培训，技术成长全靠自己
- 归属感弱，你是在"帮别人做项目"
- 工资涨幅有限，天花板明显

### 转折点：那个让我清醒的下午

入职第 3 个月，客户临时加需求，要求在 3 天内做一个数据可视化大屏。

我用 Chart.js 硬肝了 3 天，做出来的效果勉强能用。但客户看了之后说：「能不能做得像那个 XXX 大屏一样炫酷？」

我去搜了一下，发现别人用的是 Three.js + ECharts + WebGL。

那一刻我意识到：**我和真正优秀的前端，差距不只是经验，而是认知。**

我不知道有这些技术，不知道这些技术能做什么，甚至不知道该去哪里学。

## 第二章：自我觉醒

### 意识到问题

我给自己列了一个"问题清单"：

```
1. 技术栈停留在 Vue2，不了解 Vue3、React、TypeScript
2. 不会用构建工具，npm scripts 就是我的全部
3. 没有自己的技术博客和 GitHub 项目
4. 不了解前端生态（工具、框架、趋势）
5. 代码写得多，但不知道为什么这么写
6. 没有计算机基础（数据结构、算法、网络）
7. 英语阅读能力差，看不下去英文文档
```

### 制定计划

我没有辞职脱产学习（经济条件不允许），而是制定了**"业余 3 小时"计划**：

```
工作日：
- 早上 7:00 - 8:00：学算法/数据结构（1小时）
- 晚上 20:00 - 22:00：学新技术/做项目（2小时）

周末：
- 上午：系统性学习（看教程/读文档）
- 下午：实战项目/写博客
- 晚上：休息（很重要！别把自己逼太紧）
```

> **关键**：计划要可持续。不要一上来就每天学 5 小时，坚持 3 天就放弃了。

### 我的资源清单

当时用的学习资源（全免费）：

| 类型 | 资源 | 用途 |
|------|------|------|
| 视频教程 | B站（coderwhy、尚硅谷） | 系统学习 Vue3、React |
| 文字教程 | MDN Web Docs | 权威参考 |
| 刷算法 | LeetCode（每日一题） | 算法基础 |
| 技术文章 | 掘金、知乎 | 了解生态和最佳实践 |
| 英文阅读 | dev.to、Medium | 锻炼英文+了解前沿 |

## 第三章：三个关键决策

### 决策一：开始写技术博客

2023 年底，我在掘金发了第一篇技术文章。

写得很烂，阅读量只有 47。但我坚持下来了。

**写博客的好处**：

1. **倒逼学习**：要写出来，就得先学明白
2. **建立品牌**：面试时能拿出自己的技术博客
3. **加深理解**：教是最好的学
4. **积累作品集**：博客就是你最好的简历

到 2025 年底，我已经写了 50+ 篇文章，有 3 篇阅读过万。

### 决策二：转型 TypeScript

2024 年初，我做了一个重要的决定：**全面转向 TypeScript**。

这个决定一开始很难。因为：

- 公司项目全是 JavaScript，没有 TypeScript 的实践场景
- TypeScript 的类型系统一开始很劝退
- 同事都觉得"没必要，JS 够用了"

但我还是坚持了。方法很简单：**业余项目全部用 TypeScript 写**。

我用 Vue3 + TypeScript 写了一个个人博客、一个待办事项应用、一个数据可视化项目。

3 个月后回头看，代码质量提升了一个量级。TypeScript 不是"增加工作量"，而是"减少 debug 时间"。

### 决策三：从 Vue 转 React

这不是说 Vue 不好，而是**拓宽技术视野**。

学了 React 之后，我理解了：

- 组件化思想的更多可能性
- Hooks 的设计哲学
- 函数式编程在前端的应用
- 不同框架的优缺点和适用场景

**建议**：精通一门框架，但至少要了解另一门。

## 第四章：面试之路

### 准备阶段（3 个月）

2025 年下半年，我开始准备大厂面试。

**技术准备**：

1. JavaScript 基础：原型链、闭包、事件循环、Promise、async/await
2. 框架原理：Vue3 响应式原理、虚拟 DOM、diff 算法
3. 网络协议：HTTP/HTTPS、TCP、WebSocket
4. 浏览器原理：渲染流程、重排重绘、垃圾回收
5. 算法：LeetCode 热题 100 + 剑指 Offer
6. 工程化：Webpack/Vite、CI/CD、性能优化

**软技能准备**：

1. 准备自我介绍（1分钟/3分钟/5分钟 三个版本）
2. 准备项目经历（STAR 法则：Situation、Task、Action、Result）
3. 准备"你有什么想问我的"（对面试官提问也体现了你的水平）

### 投递与面试

第一次面试：一面挂了（算法题没做出来）

第二次面试：二面挂了（Vue3 原理讲不清楚）

第三次面试：一面通过 → 二面通过 → 三面挂了（系统设计题不会）

第四次面试：**全部通过，拿到 offer**

### 被问得最多的问题

1. **自我介绍** — 每次面试必问，一定要准备
2. **项目中遇到最大的挑战** — 要有具体的技术细节
3. **Vue3 的响应式原理** — 几乎每家都问
4. **HTTP 缓存机制** — 强缓存、协商缓存
5. **性能优化做过什么** — 要有数据支撑
6. **为什么想来我们公司** — 不要说"工资高"

## 第五章：入职大厂后的变化

### 好的变化

- **技术成长**：代码评审、技术分享、开源贡献，每天都能学到新东西
- **工程规范**：ESLint、TypeScript、单元测试、Code Review，代码质量有保障
- **技术视野**：接触到大规模应用的技术挑战（微前端、SSR、性能监控）
- **薪资提升**：涨幅比外包时期显著
- **学习资源**：公司有技术图书馆、在线课程、技术大会预算

### 不太适应的变化

- **节奏更快**：迭代周期短，需求变化快
- **考核压力**：有 OKR/KPI，不只是写代码
- **人际关系更复杂**：跨团队协作需要更多沟通
- **冒名顶替综合征**：觉得自己配不上这个位置（后来发现很多人都有这种感觉）

## 第六章：给外包同行的建议

如果你现在也在外包公司，我想说：

### 1. 不要觉得外包低人一等

外包是很多程序员的起点，不是终点。重要的是你从这段经历中**学到了什么**，而不是你在哪里工作。

### 2. 利用好"项目多"的优势

外包接触的项目类型丰富，这是自研公司比不了的。**每个项目都是学习机会**。

### 3. 建立技术输出

- 写技术博客（掘金、CSDN、微信公众号）
- 做 GitHub 开源项目
- 参与技术社区讨论
- 做技术分享（公司内/线上）

### 4. 不要等到"准备好了"才投简历

永远不会有"完全准备好"的时候。边投边学，面试本身也是学习。

### 5. 保持耐心

从外包到大厂，我花了 2 年多。这期间有无数次想放弃的时刻。但回头看，**每一步都算数**。

这篇文章写得比较长，因为每一个细节都是真实的经历。

我不是什么天才，也没有特殊的背景。只是一个普通的程序员，在普通的公司，做着普通的项目，但每天都在努力变得更好一点。

如果你也在经历类似的阶段，希望你记住这句话：

> **种一棵树最好的时间是十年前，其次是现在。**

开始行动吧，哪怕每天只进步一点点。
